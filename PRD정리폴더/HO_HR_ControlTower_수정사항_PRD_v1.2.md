선행 문서: `HO_HR_ControlTower_수정사항_PRD_v1.1.md`
상태: 작성 → (검토/수정) → 반영

---

## 1. 목적

홈 대시보드(핵심 포스트 노출판)의 가독성·정렬·브랜딩을 개선하고,
향후 데이터 항목 추가/수정이 가능하도록 전체 데이터 구조를 문서화한다.

---

## 2. 변경 항목

### 2-1. 워딩: 서비스명
- **현행**: 사이드바 상단 `Talent Control Tower`
- **변경**: `유통BG 인재 컨트롤타워`
- 적용 파일: `components/shell/Sidebar.tsx`, `app/layout.tsx`(metadata title)

### 2-2. 워딩: 메뉴명
- **현행**: 사이드바 NAV `포스트 매트릭스`
- **변경**: `핵심 포스트 노출판`
- 적용 파일: `components/shell/Sidebar.tsx`
- 비고: 본문 페이지 헤더는 이미 `핵심 포스트 노출판`으로 일치.

### 2-3. 법인/조직 열: 레벨 표기
- **요구**: 첫 화면 `법인/조직` 열의 `유통BG` 텍스트 아래에 `(LV6)` 표기.
- **구현**: `PostRow`에 선택적 `level?: string` 필드 추가.
  - `유통BG` 행 → `level: "LV6"`.
  - 데이터에 `level` 값이 있는 행만 조직명 아래 작은 회색 텍스트로 렌더.
  - 다른 조직도 향후 동일 방식으로 레벨 부여 가능(데이터만 추가).

### 2-4. 인사카드: 2줄 레이아웃
- **현행**: `이름 (직책/나이)`가 한 줄(inline)로 표기.
- **변경**: 1줄 = 이름(굵게), 2줄 = `직책 + 나이`(작게).
  - 과업명(상단) → 이름 → 직책/나이 순의 세로 스택.
  - 공석은 기존처럼 `공석` 표기.
- 적용 파일: `components/matrix/PostMatrix.tsx`

### 2-5. 신호 색상: 더 선명하게
- **현행**: red `#E5484D`, amber `#D9920A`, green `#1B9E6B`
- **변경(쨍한 색)**: red `#FF3B30`, amber `#FFC400`, green `#16C46A`
- 텍스트 대비: 빨강/초록 배경 → 흰색 글씨, 노랑 배경 → 진한(ink-900) 글씨 유지.
- 적용 파일: `tailwind.config.ts` (signal 토큰). KPI 카드·범례·필터에 자동 반영.

### 2-6. 행열 정렬 고정
- **문제**: 셀 카드의 가로/세로·여백이 미세하게 어긋나 표가 틀어져 보임.
- **원인**: CSS grid `1fr` 트랙의 기본 `min-width:auto` 때문에 긴 텍스트가 열 폭을 밀어냄.
- **해결**:
  - grid 템플릿을 `repeat(8, minmax(0,1fr))`로 변경(헤더/행 동일).
  - 모든 셀(빈칸 포함) 그리드 자식에 `min-w-0` 적용 → `truncate` 정상 동작.
  - 모든 셀 높이 `h-[72px]` 고정, 동일 `padding` 유지 → 가로세로·여백 통일.

### 2-7. 데이터 구조 문서화
- 전체 데이터 구조/항목을 정리한 `.md` 1개 생성.
- 포함: 매트릭스(PostRow/PostCell/색상), 내부 인재, 외부 인재, 과업 프로파일, 엑셀 양식.
- 목적: 비개발자도 보고 항목을 추가/수정할 수 있도록 위치·형식 안내.
- 산출물: `PRD정리폴더/데이터구조_가이드.md`

## 4. 수용 기준 (AC)

1. 사이드바 상단이 `유통BG 인재 컨트롤타워`, 메뉴가 `핵심 포스트 노출판`으로 표시된다.
2. 첫 열 `유통BG` 아래 `(LV6)`가 작은 글씨로 표시된다.
3. 각 인사카드가 이름(1줄)·직책/나이(2줄)로 줄바꿈되어 보인다.
4. 빨강/노랑/초록이 기존보다 명확히 선명하며, 글씨 대비가 유지된다.
5. 모든 열·행 카드의 가로/세로/여백이 동일하게 고정되어 표가 어긋나지 않는다.
6. `데이터구조_가이드.md`만 보고 새 조직/포스트/색상을 추가할 수 있다.
7. `npx tsc --noEmit` 통과.
