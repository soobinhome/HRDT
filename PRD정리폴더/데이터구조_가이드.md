## 1. 핵심 포스트 노출판 (매트릭스)

### 1-1. 구조

화면의 한 줄(행) = 조직 1개 = `PostRow`. 한 줄에 포스트 8개가 가로로 놓입니다.

```
PostRow {
  id: string          // 고유 ID (예: "row-distbg")
  area: string        // 조직명 (예: "유통BG")  ← 첫 열에 표시
  level?: string      // 조직 레벨 (예: "LV6")   ← 조직명 아래 "(LV6)"로 표시. 없으면 미표시
  current: PostCell           // 현직자 (F1)
  successors: PostCell[]      // 후계자 2명 [F2, F3]
  wings: PostCell[]           // 날개과업 5개 [1번~5번 날개]
}
```

### 1-2. 셀 1칸 = `PostCell`

```
PostCell {
  title?: string      // 과업·포지션명 (셀 윗줄, 작은 글씨)
  person?: string     // 담당자. 형식: "이름(직책/나이)"  예: "권태오(이사대우/46)"
  color: PostColor    // 신호등 색 (아래 표 참고)
  taskKey?: string    // 점수 계산용 과업 프로파일 키 (선택)
  memo?: string       // 메모 (선택)
}
```

- **이름/직책/나이 표기 규칙**: `person`을 `"이름(직책/나이)"` 형식으로 적으면,
  화면에서 자동으로 **1줄=이름, 2줄=직책/나이** 로 나뉘어 표시됩니다.
  - 예: `"권태오(이사대우/46)"` → 이름 `권태오`, 아래줄 `이사대우/46`
- **공석**: `person`을 `"-"` 또는 빈 문자열로 두면 셀에 **"공석"** 으로 표시됩니다.
- **빈 포스트**: `color: "empty"` + `title`·`person` 모두 비우면 점선 빈칸으로 표시됩니다.

### 1-3. 신호등 색상 (`PostColor`)

| 값 | 라벨 | 의미 | 셀 색 | 글씨색 |
|---|---|---|---|---|
| `red` | 빨간불 | 즉시 교체·검증 필요 | 빨강 | 흰색 |
| `yellow` | 노란불 | 지켜보기 | 노랑 | 검정(진한) |
| `blue` | 파란불 | EBG 통과 | **초록**(워딩만 파란불) | 흰색 |
| `green` | 초록불 | 안정 (호환용, blue와 동일 처리) | 초록 | 흰색 |
| `empty` | 빈칸 | 미배치 | 점선 | - |

> 색상 HEX는 `tailwind.config.ts`의 `signal` 토큰에서 변경합니다.
> 현재: red `#FF3B30` / amber(노랑) `#FFC400` / green(초록) `#16C46A`.

### 1-4. 조직(행) 추가하는 법

`lib/data/matrix.ts`의 `SAMPLE_MATRIX` 배열에 항목을 추가합니다.
헬퍼 `c(title, person, color, taskKey?)`로 셀을 간단히 만들 수 있습니다.

```ts
{
  id: "row-new",
  area: "신규조직",
  level: "LV5",            // 필요 없으면 줄 생략
  current: c("총괄", "홍길동(부장/45)", "yellow", "A인재 확보·배치"),
  successors: [
    c("", "-", "red"),                 // F2 공석
    c("F3", "김후계(과장/38)", "yellow"),
  ],
  wings: [
    c("날개과업1", "이담당(과장/36)", "blue"),
    EMPTY, EMPTY, EMPTY, EMPTY,        // 빈 날개는 EMPTY 사용
  ],
},
```

규칙:
- `successors`는 **항상 2개**(F2, F3), `wings`는 **항상 5개**로 채웁니다(빈칸은 `EMPTY`).
- `id`는 다른 행과 겹치지 않게 합니다.

### 1-5. KPI 카드 집계 규칙

홈 상단 4개 카드는 `lib/data/matrix.ts`의 `computeKPI()`가 계산합니다.
- 전체 포스트 = 색이 있고(=empty 아님) 내용이 있는 셀 수
- 빨간불/노란불/파란불 = 각 색 개수 (파란불 카드는 `blue + green` 합산)

| 필드 | 설명 | 예시 |

| `managerClass` | 경영자반 | `true/false` |
| `sproutClass` | 새싹반 | `true/false` |
| `groundExp` | 밑바닥경험 | `true/false` |

## 4. 과업 프로파일 (`TaskProfile`)

`lib/data/taskProfiles.ts`. 후보 판별기 점수(scoreTalent V8.0)에 사용.
매트릭스 셀의 `taskKey`가 이 프로파일의 `key`와 연결됩니다.

| 필드 | 설명 |
|---|---|
| `key` | 고유 키 (매트릭스 `taskKey`와 일치) |
| `label` | 과업명 |
| `group` | 과업 그룹 |
| `desc` | 설명 |
| `anchor?` | 현직자 앵커(이름) |
| `mustJob` | 필수 직군(미부합 시 패널티) |
| `competencyWeights` | 6역량별 가중치 |
| `discPrefer` | 선호 DISC |
| `strengthMust` / `strengthPrefer` | 필수/선호 강점 |
| `minYears?` | (Phase 2 예정) |

> 매트릭스 셀 `title`이 과업 프로파일 `key`와 정확히 같으면,
> 엑셀 업로드 시 `taskKey`가 자동 연결됩니다.

## 5. 엑셀 업로드 양식 (방식 B — 조직 1줄, 포스트 8개 가로)

홈 하단 **"입력 양식 다운로드"** 로 받은 `.xlsx`와 동일한 형식입니다.

| 열 | 머리글 | 내용 |
|---|---|---|
| A | 조직 | 조직명 (예: 유통BG) |
| B | 현직자(F1) | 셀 형식 |
| C | F2 | 셀 형식 |
| D | F3 | 셀 형식 |
| E~I | 날개1 ~ 날개5 | 셀 형식 |

**셀 형식**: `과업명 / 담당자(직책/나이) / 신호등`
- 신호등: `빨간불 · 노란불 · 파란불` (또는 빨강/노랑/파랑·초록)
- 공석: 담당자를 `-` 또는 공란
- 칸 전체가 비면 빈 포스트
- 색 누락인데 내용이 있으면 기본 **노란불** 처리

업로드하면 매트릭스 **전체가 교체**됩니다(확인 모달 → "전체 교체").
파싱은 **브라우저에서만** 이뤄지며 서버에 저장되지 않습니다.

관련 코드: `lib/excel.ts` (`EXCEL_HEADERS`, `buildTemplateBlob`, `parseMatrixWorkbook`).
